# Security Policy

## Supported Versions

LotteryLab is currently a pre-1.0 project. Security fixes are applied to the
latest version on the `main` branch only.

| Version | Supported |
| ------- | --------- |
| `main`  | ✅ |
| Older commits or forks | ❌ |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.

Report vulnerabilities privately using
[GitHub's private vulnerability reporting](https://github.com/ryroeu/LotteryLab/security/advisories/new).

Please include, where possible:

- A description of the vulnerability and its potential impact
- The affected file, feature, or dependency
- Steps or sample code that reproduce the issue
- Any suggested mitigation or fix
- Whether the vulnerability has been disclosed elsewhere

We aim to acknowledge reports within seven days. Confirmed vulnerabilities will
be investigated and addressed as time permits. Please allow reasonable time for
a fix before making the vulnerability public.

## Scope

Security reports may cover the LotteryLab application, command-line tools,
packaging, dependency configuration, and handling of downloaded lottery data.

Issues in third-party services or datasets are outside this project's control
unless they expose a vulnerability in the way LotteryLab integrates with them.
Statistical disagreements, feature requests, and ordinary bugs may be reported
through the public issue tracker.
